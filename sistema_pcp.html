🔗 Sincronização via GitHub

Permite que seu time veja os dados que você atualiza, sem precisar mandar arquivos. Você continua editando localmente, exporta o JSON e sobe no GitHub. O time acessa o sistema e vê os dados atualizados automaticamente.

📚 Como configurar — passo a passo
  1. Crie um repositório no GitHub (privado, gratuito) — ex: touti-pcp
  2. Suba o sistema: arraste o arquivo sistema_pcp.html pro repo
  3. Ative o GitHub Pages: Settings → Pages → Source: main branch. Seu sistema fica em seu-usuario.github.io/touti-pcp/sistema_pcp.html
  4. Exporte o JSON aqui no sistema: clique 💾 Backup no header. Renomeie pra data.json
  5. Suba o data.json no mesmo repositório (mesma pasta do HTML)
  6. Cole a URL "raw" do data.json abaixo. Pra repositório público, é tipo: https://raw.githubusercontent.com/USUARIO/touti-pcp/main/data.json. Pra privado, precisa de token (mais abaixo).
  7. Compartilhe com seu time a URL do sistema. Eles veem tudo em modo leitura.

Atualizando os dados (rotina diária):

  1. Trabalhe normal no sistema (carregar PBI, OPs em andamento, etc)
  2. Fim do dia: clique 💾 Backup → renomeie pra data.json
  3. No GitHub web, abra o repo, arraste o novo data.json em cima do antigo → commit
  4. Pronto. Time abre o sistema e vê os dados atualizados na hora seguinte.

⚠ Repositórios privados (recomendado):

Pra repositório privado, o GitHub exige um token. Mas embutir token no HTML é inseguro. Alternativa segura: use repo público mas em conta GitHub separada (só pra esse fim), sem dados sensíveis no nome dos itens. Ou contrate GitHub Pro pra ter Pages em repo privado.
